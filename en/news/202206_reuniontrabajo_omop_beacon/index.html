Reunión de trabajo para mapear el modelo estándar de OMOP CDM con las especificaciones de Beacon

  1. Home
  4. Reunión de trabajo para mapear el modelo estándar de OMOP CDM con las especificaciones de Beacon
23-06-2022


El pasado 22 de junio se realizó la primera reunión en persona, con posibilidad de participar de manera remota, del grupo de trabajo OMOP CDM & Beacon de IMPaCT-Data. El encuentro se hizo en el Parc de Recerca Biomédica de Barcelona (PRRB) con la participaron de Miquel Àngel Mayer y Juan Manuel Ramírez del UPF/IMIM; Jordi Rambla y Babita Singh del CRG; Sergi Aguiló del BSC; y Alberto M. Parra del GENyO en remoto.

El objetivo de este grupo de trabajo es mapear el modelo estándar de OMOP CDM con las especificaciones de Beacon, y de esta manera obtener una herramienta de exploración y descubrimiento común de datos clínicos y genómicos en las diferentes instituciones del proyecto IMPaCT.

El propósito principal de OMOP Common Data Model es tener un modelo estándar para bases de datos observacionales de hospitales y prácticas clínicas para su persistencia y futura explotación en usos secundarios. Gracias a este modelo común y la habilidad de OMOP CDM para trabajar con diferentes ontologías, se podrá analizar sistemáticamente bases de datos de diferente procedencia.

Por otra parte, Beacon sirve para hacer una exploración inicial de datos, totalmente anonimizados y simplificados, para ver si se encuentran en las diferentes instancias donde se hace la búsqueda, en este caso OMOP CMD. Una vez descubiertos, el usuario puede pedir el acceso completo al propietario de los datos.

La reunión fue muy productiva, ya que se definieron sobre el papel aquellas variables de Beacon y de OMOP CMD que se pueden mapear. Durante el transcurso de la reunión se comentaron los diferentes términos de Beacon uno por uno y viendo cómo relacionarlos con las variables de OMOP CMD. La principal dificultad fue entender las especificaciones de los dos modelos de datos. Por ejemplo, depende del tipo de ontología en una tabla de OMOP CMD, este valor se mapeará en distintos términos de Beacon, ya que OMOP CMD no contempla esta diferencia.

El próximo paso del grupo de trabajo será hacer un Beacon que funcione con datos sintéticos de OMOP CMD, y de esta manera comprobar si hay errores o casos límite no contemplados.

Gracias a este proyecto, y siempre cumpliendo con los principios FAIR de tener datos encontrables, accesibles, semánticamente interoperables y reutilizables, avanzaremos en el almacenamiento e intercambio de datos para un alto potencial colaborativo entre hospitales españoles.
